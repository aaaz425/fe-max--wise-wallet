### **➡️ 구현 사항 정리**
* git에 대해 학습했다 -> 동료들의 도움을 많이 받았다
```
* 나만의 브랜치 네이밍 규칙
  PR 날리는 브랜치 : week1_feat1, week1_feat2...
  작업 브랜치 : feat1_html1, feat1_css1, feat1_html2...

* PR흐름
  pr브랜치 생성 -> draft PR -> 작업브랜치 생성, 작업, 커밋 -> pr브랜치로 switch
  -> merge 작업브랜치 -> push origin pr브랜치 -> 확인 후 작업브랜치 삭제
```
